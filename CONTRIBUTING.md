# Contributing to rateLimiter

Thanks for your interest in contributing! This project follows a standard workflow:

1. Fork the repository and create a feature branch (`git checkout -b feature/my-change`).
2. Write tests for any new functionality and ensure existing tests pass.
3. Follow Go formatting conventions (`gofmt` or `goimports`).
4. Submit a pull request describing your changes, linking any relevant issues.

See the project wiki or issue tracker for more details.
